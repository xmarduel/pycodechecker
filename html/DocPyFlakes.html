PyFlakes Help

PythonCodeAnalyserApp supports PyFlakes-0.9.2. A clone of PyFlakes versionned 0.9.2.0.1 shipped withPythonCodeAnalyserApp adds some features to PyFlakes-0.9.2.

PyFlakes-0.9.2.0.1 additional features are:

  • indentation check
  • check for multiple imports on the same line
  • re import
  • comparison has no effect check
  • distinguish between parameters/variables not used

Moreover, there is the possibility to disable some warnings (especially when these are wrong, ie PyFlakes outputs false warnings) through special annotatiions in the python code.

The features in detail:

  • disable some warnings : example with "unused import" message

    if PyFlakes outputs an "unused import" warning although the import is indeed used, the user may write in its source code at the line of the import the pyflakes directive

    import <module> # pyflakes:ignore-msg#import not used

    Note: this type of false warning occurs when

    • the imported module defines statements which are executed when the import is performed (initialization statements)
    • (what else)...

  • multiple imports on the same line

    output a warning in the trivial case where more than 1 import on the same line is given

  • re- imports

    output a warning in the trivial case where a module is imported twice

  • comparison check

    statements of the form

    a == b
    a != b

    (not inside a "If" statement , or "Return" etc.) have no effect, and a warning is thus raised

  • indentation check

    statements of the form

    if 1:
            a = 1 # 8 empty spaces

    generate a "BadIndentation" message

Finally, notice that the messages of PyFlakes-0.9.2.0.1 are slightly different from those of PyFlakes-0.9.2
